Betrieb:

Der Docker-Container für die Middleware wurde erstellt und konfiguriert. Ein Multi-Stage Build wurde implementiert, der zuerst das Frontend mit Node.js baut und dann das .NET-Backend kompiliert.

Docker Compose Konfigurationen wurden für Development und Production angelegt. In der Entwicklungsumgebung werden Hot-Reload und Debug-Funktionen aktiviert, in Production läuft alles optimiert.

Die erste Version des neuen Frontends wurde mit grundlegenden Features implementiert. Es zeigt jetzt den Verbindungsstatus zu Actindo und den OAuth-Token-Status an, damit man sofort sieht ob die Systeme erreichbar sind.

Aufräumarbeiten im Repository wurden durchgeführt: alte Beispieldateien entfernt und die .gitignore aktualisiert. Das Repository ist jetzt deutlich übersichtlicher und enthält keine unnötigen Dateien mehr.

Eine README mit Deployment-Anweisungen wurde erstellt. Diese dokumentiert die Schritte für Installation, Konfiguration und Aktualisierung der Middleware.
